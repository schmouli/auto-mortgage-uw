# Database Migrations & Seed Data API

This module manages the database schema evolution via Alembic and provides endpoints to initialize seed data for development and testing environments.

## POST /api/v1/system/seed

Initializes the database with seed data (Admin, Broker, Client users, Lenders, and Products). This is typically used for environment setup.

**Permissions:** Admin only

**Request:**
```json
{
  "force_reset": false
}
```

**Response (201 Created):**
```json
{
  "message": "Seed data successfully populated",
  "details": {
    "users_created": 3,
    "lenders_created": 5,
    "products_created": 10,
    "applications_created": 1
  }
}
```

**Errors:**
- 400: Bad Request (Database already contains data, set force_reset=true to override)
- 401: Not authenticated
- 403: Permission denied (User is not an Admin)
- 500: Internal Server Error (Database connection failure)

# Database Migrations & Seed Data Module

## Overview
This module is responsible for the database lifecycle management of the Canadian Mortgage Underwriting System. It utilizes **Alembic** for schema migration management and includes scripts to populate the database with initial seed data required for the application to function (Lenders, Products) and for testing (Sample Users, Applications).

## Key Functions

### Migration Strategy
- **One Migration Per Module:** As the system consists of 20 modules, each module introduces its own migration file rather than a monolithic upgrade script.
- **Reversibility:** All migrations include a `downgrade()` method to ensure rollbacks are possible without data loss (where logically feasible).
- **Naming Convention:** Migration files are prefixed with the module name (e.g., `20260302_module_users_add_sin_index.py`).

### Seed Data Logic
The seeding process is idempotent. It checks for the existence of records before inserting to prevent duplicates on re-runs, unless `force_reset` is specified.

1. **Users:** Creates 3 default users with hashed passwords.
   - Admin: `admin@mortgage-uw.local`
   - Broker: `broker@mortgage-uw.local`
   - Client: `client@mortgage-uw.local`
2. **Lenders:** Populates the "Big 5" Canadian banks (RBC, TD, BMO, Scotiabank, CIBC).
3. **Products:** Creates 2 products per lender:
   - 5-Year Fixed Rate Mortgage
   - 5-Year Variable Rate Mortgage
4. **Sample Application:** Creates a sample mortgage application linked to the client user, including associated documents and underwriting notes.

## Usage Examples

### Running Migrations via CLI
Standard Alembic commands are used for database updates.

```bash
# Upgrade to the latest version
uv run alembic upgrade head

# Upgrade one step
uv run alembic upgrade +1

# Downgrade one step
uv run alembic downgrade -1

# Create a new migration for a specific module
uv run alembic revision -m "add_insurance_premium_column"
```

### Triggering Seed Data via API
To seed a fresh database instance via the internal API:

```bash
curl -X POST "https://api.mortgage-uw.local/api/v1/system/seed" \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"force_reset": false}'
```

## Compliance Notes
- **FINTRAC:** Seed data creation timestamps are immutable. The `created_at` field is set explicitly.
- **PIPEDA:** Default passwords in seed data are temporary and must be changed upon first login. No real PII (SIN/DOB) is included in the sample client data generated by this module.
```
