## Module Architecture

Each feature module follows the standard structure:

```
module/
├── __init__.py           # Package init
├── models.py             # SQLAlchemy ORM models
├── schemas.py            # Pydantic request/response schemas
├── services.py           # Business logic and calculations
├── routes.py             # FastAPI endpoints
├── exceptions.py         # Module-specific exceptions
├── tests.py              # Unit tests
└── design_with_stack.md  # Design documentation
```

## Design Principles

1. **Domain-Driven Design** - Modules organized by business domain
2. **Separation of Concerns** - Models, validation, logic, and routing separated
3. **Type Safety** - Full type hints throughout
4. **Audit Trail** - All changes tracked with created_by, updated_at
5. **Financial Precision** - Decimal type for all monetary values
6. **Security First** - No PII in logs, encrypted fields, JWT/OAuth
